# Проєкт "Абетка" 🦄

Освітня гра для вивчення української абетки, розроблена спеціально для дитини (Ксенії). Гра поєднує вивчення звуків/літер із механікою збирання пазлів.

## 🎯 Основна мета
Навчити дитину розрізняти голосні та приголосні літери, правильно їх вимовляти та асоціювати звук із символом через ігрову мотивацію (відкриття прихованої картинки).

## 🛠 Технологічний стек
- **Frontend:** React (TypeScript) + Vite.
- **Runtime:** Electron (для десктопної версії).
- **Стилізація:** Vanilla CSS (App.css).
- **Ефекти:** `canvas-confetti` для святкування перемог.
- **Аудіо:** Локальні HTML5 Audio (`.mp3`) + Fallback на Web Speech API (`uk-UA`).

## 🏗 Структура проєкту
- `src/data/letters.ts` — серце логіки даних:
  - Списки `VOWELS` та `CONSONANTS`.
  - Функції `speakLetter`, `playRandomFeedback` та `initSpeech`.
  - **Важливо:** Для Electron шляхи до аудіо мають починатися з `./audio/`.
- `src/App.tsx` — головний компонент гри:
  - Керування станом (рівень, очки, відкриті шматочки пазла).
  - Логіка генерації раундів та перевірки відповідей.
- `public/audio/` — звукові активи (літери, похвала `praise_X.mp3`, помилки `error_X.mp3`).
- `public/images/puzzles/` — картинки для рівнів (назви `1.jpg`, `2.jpg` тощо, до 20).

## 🕹 Ігрова механіка
1. **Налаштування:** Вибір режиму (Голосні/Приголосні/Все разом) та складності (кількість варіантів літер).
2. **Прогресія рівнів:**
   - 20 рівнів. З кожним рівнем сітка пазла стає складнішою:
     - Рівень 1: 2x2 (4 шматки).
     - Рівень 20: 5x4 (20 шматків).
3. **Ігровий цикл:**
   - Система озвучує цільову літеру.
   - Дитина обирає варіант.
   - **Правильно:** Відкривається випадковий шматочок пазла + конфеті + похвала.
   - **Помилка:** Якщо вже є відкриті шматки, один випадково "ховається" (штраф за помилку, але не більше одного разу за раунд) + фраза підбадьорення.
4. **Перемога на рівні:** Коли всі шматочки відкриті, відтворюється фінальний звук (`praise_20.mp3`), салют, і з'являється кнопка "Далі".

## ⚠️ Технічні нюанси та обмеження
- **Шляхи до ресурсів:** В Electron (особливо в збірці `.asar`) критично використовувати відносні шляхи `./audio/` та `./images/`.
- **Аудіо-контекст:** Браузери блокують звук до першої взаємодії. Функція `initSpeech` викликається при натисканні "Почати пригоду".
- **Генерація випадковості:** Використовується історія (`lastTargetLetters`, `lastPraiseIndices`), щоб уникнути повторення однакових літер або фраз кілька разів поспіль.

## 📝 Правила розробки
- Дотримуватися існуючого стилю іменування (camelCase для змінних, PascalCase для компонентів).
- Не видаляти синтез мовлення (SpeechSynthesis) — це важливий резервний варіант, якщо MP3 не завантажився.
- Кожне нове зображення для пазла має бути перевірене на коректність відображення в сітці.
